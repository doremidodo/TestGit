* 용어
	pull - git 저장소 서버에서 내 컴퓨의 저장소로 내려받는 것
	commit - 추가, 수정, 삭제 변경사항을 1개의 버전으로 기록 (로컬)
	push - 로컬의 정보를 git 저장소 서버로 올리는 것
	=> * 가능하면 서버에 올릴 때 (commit & push) 한 동작 단위로 처리
	Branch - 서버에 올라오는 버전들을 브런치 (기본 master)
		   - 내 컴퓨터 브런치 - 로컬 브런치, 서버 브런치, 리모트 브런치
	tag - 특정 버전을 찾기 쉽게 이름을 지정
	Merge - 병합, 여러명이 작업한 코드를 합치기
